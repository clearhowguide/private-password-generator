# 🤝 Contributor Code of Conduct

## 🌿 Our pledge

We want Private Password Generator to be a welcoming, respectful, and useful project for everyone.

As contributors and maintainers, we pledge to make participation in this project a harassment-free experience for everyone, regardless of experience level, background, identity, appearance, disability, nationality, race, age, religion, gender identity, sexual orientation, or personal beliefs.

---

## ✅ Our standards

Examples of positive behavior include:

- using welcoming and inclusive language;
- being respectful of different viewpoints;
- accepting constructive feedback gracefully;
- focusing on what is best for the project and its users;
- showing empathy toward other contributors;
- helping beginners understand technical decisions;
- improving privacy, accessibility, security, and clarity.

Examples of unacceptable behavior include:

- harassment, threats, insults, or personal attacks;
- sexual language or imagery;
- trolling or repeated disruptive behavior;
- publishing private information without permission;
- dismissive or hostile comments toward beginners;
- intentionally misleading users about privacy or security;
- adding features that collect sensitive information without clear consent;
- any behavior that would reasonably be considered inappropriate in a professional setting.

---

## 🔐 Project-specific expectations

Because this project is about password generation, password safety, and privacy, contributors must be especially careful.

Do not:

- ask users to share real passwords;
- include real passwords in issues, pull requests, screenshots, logs, examples, or documentation;
- promote unsafe security advice;
- add features that silently collect generated passwords;
- add features that store generated passwords in browser storage;
- add network calls that send generated passwords away from the user's browser;
- use `Math.random()` for password generation;
- weaken or remove the `crypto.getRandomValues()` generation model without strong security justification;
- make exaggerated claims such as "impossible to hack," "100% secure," or "unhackable."

Do:

- keep examples fictional;
- explain limitations honestly;
- respect user privacy;
- prioritize clarity and safety;
- preserve the privacy-first promise of the project;
- prefer simple, understandable security explanations for non-expert users.

---

## 🛡️ Enforcement

Project maintainers may remove, edit, or reject comments, commits, issues, pull requests, and other contributions that do not follow this Code of Conduct.

Maintainers may also temporarily or permanently ban contributors for behavior that is harmful, abusive, or not aligned with the project values.

---

## 📩 Reporting

If you experience or notice unacceptable behavior, please report it through the official ClearHowGuide contact page:

<pre><code>https://clearhowguide.com/en/contact/</code></pre>

Please include:

- what happened;
- where it happened;
- any relevant links;
- screenshots if appropriate;
- whether you want your report to remain private.

Do not include real passwords, generated passwords, credentials, API keys, tokens, or sensitive account information.

---

## 🧭 Scope

This Code of Conduct applies within:

- GitHub issues;
- pull requests;
- discussions;
- code review;
- project documentation;
- related project communication spaces.

It may also apply when someone is representing the project in public spaces.

---

## 🎯 Goal

The goal is simple:

Keep the project useful, respectful, privacy-first, secure, and safe for everyone.
