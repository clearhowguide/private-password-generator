# Security Policy

## Reporting a vulnerability

Please do not report security vulnerabilities in public issues.

Use the official ClearHowGuide contact page:

<pre><code>https://clearhowguide.com/en/contact/</code></pre>

Do not include real passwords, account credentials, API keys, tokens, or private secrets.

## Scope

Security reports may include:

- accidental password storage;
- accidental network transmission;
- unsafe random generation;
- privacy regressions;
- dependency concerns;
- documentation that could mislead users.

## Important note

This project is educational. It does not guarantee that any generated password is impossible to crack.
